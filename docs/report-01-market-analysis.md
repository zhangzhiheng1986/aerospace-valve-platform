### 1.2 市场空白识别

**核心发现**：全球没有一个产品同时满足以下条件：
1. ✅ 覆盖全谱系流体力学（基础→可压缩→水锤→FSI→气体管线）
2. ✅ 免费Web访问 + 深色主题 + 中英双语
3. ✅ 公式透明（带LaTeX推导） + 交互式计算
4. ✅ 工业标准合规（ISA/ISO/ASME/API）
5. ✅ 航空航天专用流体数据

**我们的定位** = Engineering ToolBox的广度 + Omni Calculator的UX + 航空航天领域的深度

## 二、目标用户画像

### 2.1 核心用户群（按市场规模）

| 用户群 | 规模估计 | 使用频率 | 核心需求 | 付费意愿 |
|--------|----------|----------|----------|----------|
| **航空航天阀门工程师** | ~50K全球 | 日活 | 合规计算、标准符合性 | 高（$50-200/年） |
| **机械/化工工程师** | ~2M全球 | 周活 | 快速参考、设计验证 | 中（$20-50/年） |
| **高校师生** | ~3M全球 | 学期性 | 教学/学习辅助 | 低（免费优先） |
| **石油天然气管道工程师** | ~100K全球 | 日活 | 管线/气体计算 | 高（企业预算） |
| **液压系统工程师** | ~200K全球 | 周活 | 液压缸/泵/蓄能器 | 中 |
| **科研人员** | ~500K全球 | 不定 | 量纲分析、可压缩流 | 低-中 |

### 2.2 用户旅程痛点

1. **找公式难** — 目前必须手动浏览13个分类，无搜索/收藏
2. **单位混乱** — 仅支持SI，美国/英制用户无法使用
3. **无法对比** — 改变参数后无法对比前后结果
4. **无出口** — 结果不能保存/导出/分享
5. **无上下文** — 缺少公式适用范围、限制条件、参考文献

## 四、战略建议：三阶段路线图

### Phase 1: Foundation（当前→3个月）🔴最高优先级
**目标**: 巩固核心计算能力，修复已知问题，建立用户信任

| 优先级 | 功能 | 用户价值 | 技术难度 |
|--------|------|----------|----------|
| P0 | 修复公式category缺失问题（135条） | 消除导航混乱 | 低 |
| P0 | 新增单位系统切换 (SI/Imperial/US) | 打开美国/英制市场 | 中 |
| P0 | 结果导出 (PDF/Excel/CSV) | 企业合规必需 | 中 |
| P0 | 公式搜索 & 过滤 | 核心UX瓶颈 | 低 |
| P0 | 输入端流体属性库集成 | 简化输入（选流体自动填属性） | 中 |

### Phase 2: Differentiation（3→6个月）🟡关键差异化
**目标**: 建立与竞品的明显区别

| P1 | 交互式图表 (Chart.js) | 可视化结果 | 中 |
| P1 | 侧边对比模式 | 参数优化场景 | 中 |
| P1 | 收藏/历史记录 (localStorage) | 用户留存 | 低 |
| P1 | 参考文献 & 公式适用范围 | 学术/工程可信度 | 低-中 |
| P1 | 批处理计算 | 效率工具 | 高 |
| P1 | 管道网络串并联求解器 | 复杂工程场景 | 高 |

### Phase 3: World-Class（6→12个月）🟢世界级竞争力
**目标**: 达成愿景 — 世界级流体力学超级计算器

| P2 | 非牛顿流体模型 | 化工/食品/生物工程 | 高 |
| P2 | 多相流基础模型 | 石油/化工 | 高 |
| P2 | OpenAPI文档 | 开发者生态 | 低 |
| P2 | PWA离线支持 | 现场工程 | 中 |
| P2 | ISA/IEC标准阀门选型 | 过程控制 | 中-高 |
| P2 | 空化侵蚀预测模型 | 高价值细分 | 中 |
| P2 | 流体混合规则 | 过程工程 | 中 |
